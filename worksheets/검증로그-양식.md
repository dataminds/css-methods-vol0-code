# 검증 로그 양식 (책 1장 §1.7 · 부록 B)

> 모든 실습의 마지막 산출물. 4줄이면 된다. AI가 짜 준 코드를 확인 없이 쓰지 않았다는 증거이자, 몇 달 뒤의 나를 위한 기록.

## 양식

**시킨 것**: (프롬프트 요지 ; 4가지를 못박았는가 = 상태·규칙·횟수·재현[씨앗])

**확인한 것**: (검증 3법 중 실제로 한 것과 결과 ; 최소 2법)
- 손 검산: 출력 일부를 손으로 재계산해 대조 (작은 경우로 줄여서)
- 극단값 시험: 답이 뻔한 입력 → 뻔한 답이 나오는가
- 재실행 시험: 같은 씨앗(73) 두 번 → 동일한가

**판단**: (코드가 내 규칙을 맞게 구현했는가 ; 아니면 무엇을 고쳤는가)

**남은 의심**: ("없음"이라 쓰기 전에 한 번 더)

## 자가 점검

- [ ] "잘 돌아갔다"만 쓰지 않았다 (실행 ≠ 검증)
- [ ] "읽어 보니 맞는 것 같다"만 쓰지 않았다 (읽기 ≠ 검증)
- [ ] AI에게 "맞아?"라고 물은 답을 검증으로 치지 않았다
- [ ] 수치에 씨앗·파라미터를 붙였다
